The `.github/workflows/ios-build.yml` file tries to run:
`./gradlew :composeApp:embedAndSignAppleFrameworkForXcode`

This task is specifically generated by the Kotlin Multiplatform plugin for Xcode to use during its build phases. It expects environment variables from Xcode like `CONFIGURATION`, `PLATFORM_NAME`, and `ARCHS`. Because we are running it from the command line outside of `xcodebuild`, it fails with:
`error: Could not infer iOS target architectures. Make sure to build via XCode (directly or via Kotlin Multiplatform Mobile plugin for Android Studio)`

Since the next step in the workflow actually runs `xcodebuild`:
```yaml
    - name: Build iOS App (Simulator)
      run: |
        xcodebuild -project iosApp/iosApp.xcodeproj \
          -scheme iosApp \
          -sdk iphonesimulator \
          -destination 'platform=iOS Simulator,name=iPhone 15' \
          clean build \
```
The `xcodebuild` command *will* automatically trigger the Gradle build (usually via a Run Script phase in Xcode that calls `./gradlew :composeApp:embedAndSignAppleFrameworkForXcode`). Therefore, the standalone Gradle step is completely unnecessary and is the exact cause of the failure.

Wait, is the standalone Gradle step needed for something? Some projects use a workaround like `CONFIGURATION=Debug PLATFORM_NAME=iphonesimulator ARCHS=arm64 ./gradlew :composeApp:embedAndSignAppleFrameworkForXcode`, but usually, the Xcode project is already configured to do this.

If it does, we can just remove the `Build KMP Framework` step.
